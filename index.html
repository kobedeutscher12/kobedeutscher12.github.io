Projects

Chess

Motivation of the project:
I chose to do this project because it allowed my to explore the depths of object orriented coding. Chess is a great example of being able break a large problem down into its most basic pieces. From the outside the game is extremely complecated but I got to take that huge project and break it down into its individual object.

Functional requirements/what the project does:
This project implements a basic version of two player chess were the user tells the board which piece they would like to move and where. The primarly logic myself and my team created is to check if the user can legally move a piece and if not it tells them they cannot and allows them to reenter the move. This project appeared relatively simple at first but as you dive into the numerous aspects of it, it gets very complex.

Project design: The design for this project was to break it into three unique layers. One being the GUI, two being the board which would be an 8x8 array of arrays that stored pieces or null if there was no piece, and then three being the pieces which all inherite from an abstract piece type and are able to store their location, update their location and return possible moves.

Implementation:
For the project design I took on the brunt of the logic for it. I set up both a class called location which the pieces would have that returns a collumn and a row. The pieces use this to determine where they are in an 8x8 matrix of arrays that is the board. Then each piece creates a method with in them that is unique to the type of piece they are which returns all possible squares they can move to in an empty board. The board class which keeps track of pieces in teh 8x8 array (null if there is no piece and the piece located there if there is a piece) then determines from the possible moves what is a valid move. The valid move simply checks if there are pieces blocking it from getting to the input square or if the square they are going to has a piece of their type on it. My teammate Kent used these functions when setting up the GUI to allowed the GUI to function as seen in the video where it displays the possible moves for a selected piece.
